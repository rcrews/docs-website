JVMDebugSpec

(Appears on: NifiRegistrySpec, NifiSpec)

JVMDebugSpec specifies the debug configuration to be applied to the target JVM. Note: If additional Portforwarding is applied to the service, then the developer environment can use ‘localhost’ as remote address. e.g.: kubectl port-forward svc/nifi-sample 8000:8000

Field Description
suspend
bool		 (Optional)

Suspend if set to true, instructs the target JVM to suspend it’s boot process, until a debug client connects. Beware: Delaying a JVM start can cause the underlying pod’s liveness-probe to kick in, and terminate the pod. To avoid that, - for debugging purposes - one could specify: statefulset: livenessProbe: failureThreshold: to an arbitrary big number.
port
int32

Port is the debug port on which the target JVM will listen.

OpenIDSpec contains configuration for client certificate authenticated environments

Field Description
discoveryURL
string

DiscoveryURL is the URL for the desired OpenId Connect Provider (http://openid.net/specs/openid-connect-discovery-1_0.html)
connectTimeout
string		 (Optional)

ConnectTimeout specifies the connect timeout when communicating with the OpenId Connect Provider
readTimeout
string		 (Optional)

ReadTimeout specifies the read timeout when communicating with the OpenId Connect Provider
clientSecretName
string

ClientSecretName is the name of the Kubernetes secret that contains the clientID and clientSecret
JWSAlgorithm
string		 (Optional)

JWSAlgorithm is the preferred algorithm for for validating identity tokens. If this value is blank, it will default to RS256 which is required to be supported by the OpenId Connect Provider according to the specification. If this value is HS256, HS384, or HS512, NiFi will attempt to validate HMAC protected tokens using the specified client secret. If this value is none, NiFi will attempt to validate unsecured/plain tokens. Other values for this algorithm will attempt to parse as an RSA or EC algorithm to be used in conjunction with the JSON Web Key (JWK) provided through the jwks_uri in the metadata found at the discovery URL.
